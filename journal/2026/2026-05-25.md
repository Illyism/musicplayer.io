### 2026-05-25 ~14:15 UTC — Major-version breaking changes (research + fixes)

Looked up breaking changes for the big semver jumps:

#### `lucide-react` 0.x → 1.x

- Brand icons removed (Twitter, Facebook, GitHub, etc.)
- Icon renames (shape-first naming, e.g. `XCircle` → `CircleX`)
- Codemod available: `npx @lucide/codemod@latest migrate-from-0.x`

**Fix applied:** `components/share-modal.tsx` — replaced `Twitter` / `Facebook` lucide imports with inline SVG components. Rest of codebase uses non-brand icons; no other renames needed.

#### `react-resizable-panels` 3.x → 4.x

- `PanelGroup` → `Group`, `PanelResizeHandle` → `Separator`
- `direction` → `orientation`
- `onLayout` → `onLayoutChange`
- CSS: `data-panel-group-direction` → `aria-orientation`

**Fix applied:** `components/ui/resizable.tsx` — migrated wrapper to v4 primitives; kept `direction` prop as alias for `orientation` so existing pages (`app/page.tsx`, `app/r/[slug]/page.tsx`) did not need changes.

#### `shadcn` 3.x → 4.x

- CLI-only breaking changes (`create` → `init`, new `--dry-run` / `--diff` / `--view` flags, preset/monorepo support)
- No runtime app code changes required for existing installed components

#### `@base-ui/react` 1.0 → 1.5

- Only OTPField breaking change (`sanitizeValue` → `normalizeValue`); project only uses Combobox — no code changes needed.

**Verification:** `bun run check-types` passed after fixes.
